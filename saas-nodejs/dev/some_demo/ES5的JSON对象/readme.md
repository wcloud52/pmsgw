#**JSON**对象

###1、stringify()

原型：

    JSON.stringify(obj, callback, spaceNumber);

说明：

    1.1 将**对象**、**数组**转换为JSON字符串；

    1.2 也可以转换boolean类型（true变为字符串“true”）、
        number类型（1变为字符串“1”）、
        string类型（字符串“1”变为字符串“”1“”，注意，这个多了一对引号，外面的一对引号表示是字符串，里面的一对引号是修饰1的）、
        null类型（null变为字符串“null”）、
        
    1.3 不能转换undefined（返回undefined）
        函数（返回同上）；

    2.1 第一个参数obj表示被序列化的对象，具体参照1.1~1.3
    
    2.2 第二个参数表示回调函数，即用于处理第一个对象的函数。
        他是递归的执行的，有两个参数；
        第一个参数key是key，第二个参数val是值val；
        this指向的是当前被递归的对象。
        正常情况下，初始的key是空字符串，val是stringify的参数1；
        然后遍历stringify的参数1的各个属性，该怎么转就怎么转，如果遇见val是对象，再遍历val的各个属性

    2.3 第三个参数表示空白符，默认是0，即字符串在一行。
        当大于0的时候（假如为n），每个key、val组合单独一行，并缩进n个空白符。
        同级的缩进保持一致，如果某个属性是对象，则该属性的key、val组合相对于该对象的缩进程度，再缩进n个空白符
       
    3.  总结：
        简单来说，由于对象是树形的，因此从根往叶逐个递归遍历序列化目标。
        
        一般来说，不需要有第二、三个参数。
        当有第二个参数时，需要有返回值，返回值是val（即使用这个的原因是val可能被处理）
        同时，如果返回的val是个对象，那么该val则将被递归遍历处理（即使原val不是对象，但因为对象会被递归，因此val会被递归遍历）
        
        当有第三个参数时，一般是为了格式化显示。

###2、parse()

原型：

    JSON.parse(obj, callback);
    
说明：

    1. 用于处理被序列化后的对象，
    2. 第一个参数是被反序列化的对象，第二个参数是在反序列化对象结束后，再遍历结果，然后对值进行处理的回调函数；
    3. 一般情况下，只需要第一个参数即可
    4. 注意，第二个函数在执行前，已经反序列化结束了，然后遍历反序列化结束后返回的对象
    5. this是当前遍历的对象，key是当前遍历的属性，val的当前遍历的属性的值，返回值是经过处理后当前所遍历的属性的值；
    6. 直接返回val则没变化；
    7. 返回undefined，则该属性和值不会被继续递归（假如值是一个对象的话），也不会添加到最后JSON.parse()返回的值之中。
